Unity+Git

1.使用原因

最初使用的原因是，平常在家是用Windows开发，有时候出门只能带mac，需要两边同步，在测试过程中不小心把项目弄崩溃了，好家伙没备份直接一锅端掉。此外，以后在涉及多人协同开发时也很有用。

2.仓库选择

Github，较为稳定，且不会出现什么特别情况，但需要科学上网，胜在安全性，像之前拿gitee作图床结果突然就被锁了。

4.后续

直接使用GitHub Desktop进行个人项目管理即可，非常方便

5.注意事项

1.Github网络可能会不稳定，也可以选择gitee

2.unity中好像是有GitHub插件的，但我更喜欢软件一点

3.觉得麻烦可以选择unity主推的PlasticSCM，契合度更高，在编辑器中直接可以使用

4.最好都使用相同的unity版本，尤其是跨系统，例如我在Windows上使用的是2021.3.11.f1c2，mac上的2021.3.11.f1居然都无法打开报错，无奈均改成2021.3.15.f1c1
