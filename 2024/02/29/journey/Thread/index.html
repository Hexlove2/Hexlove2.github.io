5.TCP原理

1.缓冲区

  • I/O缓冲在每个tcp套接字中单独存在
  • I/O缓冲在创建套接字时自动生成
  • 即使关闭套接字也会继续传递输出缓冲中遗留的数据
  • 关闭套接字将丢失输入缓冲区中的数据

6.UDP

1.一般特性

  • UDP传输中，双方调用I/O次数要一致
  • 如果不提前连接好套接字的目标地址端口，那么每次使用sendto函数时都会反复注册与删除目标的地址信息，十分耗费时间，长时间使用udp最好提前注册好

7.TCP半关闭

1.shutdown(int sock, int howto)函数

howto:

SHUT_RD: 断开输入流

SHUT_WR: 断开输出流

SHUT_RDWR: 输入输出全断

why：

更好地控制程序执行，及时释放资源，使其分配更加灵活

8.DNS系统

作用：通过连接DNS服务器获取到域名对应的IP地址

1.gethostbyname( const char * hostname)及hostent结构体

gethostbyname可以通过域名字符串返回一个带有ip地址信息的结构体hostent

9.套接字的各种选项

1.通过getsockopt( )及setsockopt( )可查看及修改套接字的一些状态

EG:

**getsockopt(int sock, int level, int optname, void optval, socklen_t optlen);

2.SO_REUSEADDR地址再分配

主动断开连接的一方一般会有个time-wait时间，并且是基于套接字的，因为要防止最后的FIN消息确认ACK成功送达对方，否则对方会一直重复发送FIN消息，直接到达重复次数限制主动断开。如果是客户端那还好，因为客户端可以一直动态分配端口号，用的套接字也不一样，但服务器端只有那么多端口号，因为ip变不了，在网络较差的环境下time-wait被重复触发可能会造成端口号耗尽，因此需要一定的优化。使用也比较方便，设为1即可允许再分配。

3.Nagle算法

用于防止数据包过载的算法

Nagle算法可以使得网络负载下降，但传输速度也会变慢

大文件数据最好别用Nagle算法

2.通过fork函数创建进程

fork函数，头文件为<unistd.h>，成功时返回子进程的PID，失败返回-1

父进程必须请求子进程传递的值，要么exit要么return，不然子进程会成为僵尸进程，因为操作系统会一直保存这些进程直到父进程销毁或主动请求到子进程传递的值。

2.waitpid函数(不会阻塞，没有终止的子进程则返回0)
