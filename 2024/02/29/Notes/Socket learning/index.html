5.TCP原理

1.缓冲区

  • I/O缓冲在每个tcp套接字中单独存在
  • I/O缓冲在创建套接字时自动生成
  • 即使关闭套接字也会继续传递输出缓冲中遗留的数据
  • 关闭套接字将丢失输入缓冲区中的数据

6.UDP

1.一般特性

  • UDP传输中，双方调用I/O次数要一致
  • 如果不提前连接好套接字的目标地址端口，那么每次使用sendto函数时都会反复注册与删除目标的地址信息，十分耗费时间，长时间使用udp最好提前注册好

7.TCP半关闭

1.shutdown(int sock, int howto)函数

howto:

SHUT_RD: 断开输入流

SHUT_WR: 断开输出流

SHUT_RDWR: 输入输出全断

why：

更好地控制程序执行，及时释放资源，使其分配更加灵活

9.套接字的各种选项

1.通过getsockopt( )及setsockopt( )可查看及修改套接字的一些状态

EG:

**getsockopt(int sock, int level, int optname, void optval, socklen_t optlen);

2.SO_REUSEADDR地址再分配

主动断开连接的一方一般会有个time-wait时间，并且是基于套接字的，因为要防止最后的FIN消息确认ACK成功送达对方，否则对方会一直重复发送FIN消息，直接到达重复次数限制主动断开。如果是客户端那还好，因为客户端可以一直动态分配端口号，用的套接字也不一样，但服务器端只有那么多端口号，因为ip变不了，在网络较差的环境下time-wait被重复触发可能会造成端口号耗尽，因此需要一定的优化。使用也比较方便，设为1即可允许再分配。

3.Nagle算法

Nagle算法可以使得网络负载下降，但传输速度也会变慢

大文件数据最好别用Nagle算法

14.多播与广播

1.多播

多播基于UDP，只会发送一次数据，组内的客户端均可收到数据，需要多个路由器复制数据包传递给多个主机。多播组为D类ip地址，224.0.0.0 - 239.255.255.255。有些路由器不支持多播，可用隧道技术，但不是程序员要关心的问题。

路由，TTL，加入组的方法

TTL每经过一个路由器就会-1，当生存时间为0时自动销毁，过大影响网络流量，过小无法传递到目标，可以通过设置套接字的函数来设置，在tcp或udp的首部中，会有一个TTL，同样加入多组播也通过设置套接字，要用到一个新的结构体：ip_mreq。

15.套接字和标准I/O

1.标准IO函数的优缺点

1.具有良好的移植性

2.可利用缓冲提高性能

由于数据包中首部的存在，缓冲可以有效减少数据包的发送数量，效率更高，标准IO函数会有单独的数据缓冲区，而创建套接字时，也会生成套接字专用缓冲区，一共两层，套接字缓冲最接近实际网络。

若想实现一个复制文件的操作，并且是一个大小不小的文件，例如300MB的一个txt

这时使用read&write函数，则需数分钟以上

而使用标准IO函数，如fgets&fputs函数，则片刻即可

标准IO函数的缺点

如果没缺点谁还不用这个呢

1.不容易进行双向通信

2.有时频繁调用fflush函数

3.需要以FILE结构体指针的形式返还文件描述符

平常的文件描述符，就是个int型的数，但标准IO函数必须用FILE格式的指针，当然这是可以转化的

2.使用标准IO函数

fdopen函数：将文件描述符转为FILE指针

fileno函数：将FILE指针转为文件描述符

优点：

1.可以分开去编写输入和输出函数，易于开发维护

2.与输入无关的输出操作速度得以提高

在使用FILE指针去简单的fclose了写指针则会关掉整个套接字，此时虽然读指针没有关闭但已经无法接受信息了，因此这种方式并不是半关闭

2.文件描述符的复制和半关闭

上面问题的原因是，实际上读写指针只是名字上有区分，实际上均指向同一个文件描述符，最好的方式是，在IO分离前，就复制一份文件描述符，两个文件描述符均指向同一个套接字，这时再分别让读写指针去分别指向他们即可，但此时关闭其中一个仍然不是半关闭，因为套接字本身并没有被销毁，无法发送EOF信息。

解决方法：其实也很简单，就是用fileno函数，返回了文件描述符，再用shutdown函数即可

18.多线程服务器端的实现

1.线程的概念

本章内容较多，也是因为确实比较重要吧，要加深理解，多加应用

多进程服务器的缺点：

1.创建进程的开销

2.进程间的数据交换较为繁琐

3.频繁的上下文切换，十分消耗资源

上下文切换，需要反复访存IO，十分消耗时间，保存上下文，更新PCB，切换上下文之类的

线程创建切换上下文更快，并且最重要的是共享内存资源

进程需要数据区，堆区，栈区

而线程共享数据区，堆区。只有单独的栈区，确保各自的函数执行流

进程相当于在操作系统里创建了各种执行流，而线程相当于在进程里创建了多种执行流
