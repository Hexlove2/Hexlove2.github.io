  1. C++ 中的虚函数和纯虚函数有什么区别？说一下虚函数表？

​ • 虚函数：允许子类重写的方法。在基类中声明时用 virtual 关键字。

​ • 纯虚函数：强制子类必须重写的方法。在基类中声明时用 = 0。

关于虚函数表：

​ 首先只要类有虚函数定义，就会有虚函数表，并且会隐含一个成员虚函数表指针，大小看操作系统位数，这个表在编译链接时就已经确定好了，并且同一个类所有对象共享，而在实际运行时，在用基类指针管理派生类时，自然使用的是派生类本身的 vptr，就达到了多态的效果

  1. 如何避免内存泄漏？请提供示例代码。如何排查内存泄露？

​ ・ 使用智能指针（如 std::unique_ptr 和 std::shared_ptr）来自动管理内存。

​ ・ 确保每个 new 都有对应的 delete。

内存泄露的排查可以使用 Valgrind 检测泄露和非法内存访问等问题

  1. 什么是智能指针？如何使用？有哪几种常用智能指针？需要注意的点是？实现的原理是？

​ ・ 智能指针是 RAII 的实现，用于自动管理资源，避免内存泄漏。

常见的有 unique_ptr, shared_ptr, weak_ptr，需要注意的点是 shared_ptr 的相互引用问题，可能导致引用计数无法为 0 最终资源无法正常释放

原理上使用了 RAII 的思想，将资源托管，析构函数中检测到一定条件才会释放资源，同时实现一些运算符，达到指针的功能

同时 unique_ptr 直接禁止了拷贝构造和直接赋值，除非用右值移动构造或赋值

shared_ptr 则需要加入线程安全的引用计数器，在析构，拷贝赋值等情况下要进行处理

  1. C 语言中的指针和引用有什么区别？

# 2.Linux 系统

# 1.linux 的内核是什么？目的是？

首先内核是利用 c 语言和一点汇编语言实现的一个软件核心，其作用是管理和分配所有的资源，实现的功能有内存管理，文件管理，进程管理，设备管理等

# 2.linux 的体系结构以及为什么要这样分配？

linux 系统分为用户空间和内核空间，用户空间给与用户使用例如调用一些 C 库，内核空间主要为 3 个部分，一个是系统调用接口，一个是内核，一个是平台架构的代码

这样做的原因最重要的就是保证系统的安全，因为有些内容数据等是不可能让用户去随意访问的，假如出现什么 bug 了就完蛋了，同时这也对应了现代 cpu 的两种工作模式

# 3.Linux 整个开机的流程简单说一下

1. 第一步就是加电自检，确认各个硬件，例如内存 cpu 显卡等

2. 第二步则是加载引导，常见的有 GRUB 加载器，从启动盘加载 boot loader 到内存，最后把内核也加载到内存里，后面再启动

3. 之后就是内核初始化，cpu 会处理内核，解压内核映像，CPU 初始化，内存管理初始化，必要驱动加载，挂载根文件系统

4. 切换到根文件系统，运行第一个 init，pid 为 1

5. 用户登录，此时系统初始化完成，用户可以登录

6. 登录完成，使用用户空间

# 5. 关于文件描述符以及 read，write 系统调用

在 linux 系统里，万物都可以被理解为文件，比如本地文件，套接字，设备，管道，还有信号量，最后都能用一个文件描述符来代替，而每个进程里都有个文件描述符表，记录了 fd 以及对应的类型还有指向某个缓冲区的指针

像 tcp 和 udp 读一次缓冲区就空了所以要注意多进程的竞争问题，而 udp 因为是不可靠传输所以一定要及时读取，否则丢失数据不负责

# 6. 关于 swap 交换空间

swap 交换空间在安装系统时可以自己分配，主要用途是当内存不够时临时作为虚拟内存，虽然速度会变慢但不至于杀死某些进程

# 7. 关于文件系统

使用的是 B + 树，而进程调度相关则使用红黑树，保证增删效率

文件夹本身也是一种特殊文件，而每个文件都有对应的 inode，记录了很多信息，包括实际存储的位置，创建日期，文件格式等

目录作为特殊文件映射表实现通过文件名去映射表中找到 inode 的 index，inode 本身是存在数组中，一个磁盘分区会有一个 inode table，在初始化时会根据磁盘大小等分配 inode 数量，关于空闲的 inode，会采用位图或者一个队列来存放空闲的 inode 编号

# 8. 软硬链接问题

硬链接就像是一个索引，指向了同一个 inode，并且引用计数 + 1

软链接则是一个独立的文件有自己的 inode，但文件的内容是原文件的路径名，这样就找到了对应的 inode，但原文件被删的话就会失效

注意硬链接无法链接文件夹但软链接可以

# 2. 多进程与多线程

# 1. 线程池

基础概念问题

  1. 什么是线程池？

​ ・ 答：线程池是一种线程管理技术，它维护了一定数量的线程，线程池中的线程可以复用来执行多个任务，从而避免频繁创建和销毁线程的开销。

  1. 线程池有哪些优点？

​ ・ 答：线程池的优点包括：

​ ・ 提高性能：通过复用线程，减少线程创建和销毁的开销。

​ ・ 提高响应速度：当任务到达时，无需等待新线程创建即可立即执行。

​ ・ 控制并发量：通过限制线程池中的线程数量，防止系统过载。

实现细节问题

  1. 如何实现一个基本的线程池？

​ ・ 答：实现基本线程池的步骤包括：

​ ・ 创建一个任务队列。

​ ・ 创建固定数量的线程，每个线程从队列中取任务执行。

​ ・ 使用锁和条件变量管理任务队列的访问和线程的等待 / 唤醒。

  1. 解释线程池中的核心线程数和最大线程数。

​ ・ 答：核心线程数是线程池在空闲时保留的线程数量，这些线程会一直存在。最大线程数是线程池中允许存在的最大线程数量，当任务超过核心线程数时，线程池会创建新线程，直到达到最大线程数。

  1. 什么是任务队列？线程池如何使用任务队列？

​ ・ 答：任务队列用于存储等待执行的任务。当所有线程都在忙碌时，新任务会被放入队列中等待。当线程完成任务后，会从队列中取出下一个任务执行。

应用场景问题

  1. 线程池适用于哪些场景？

​ ・ 答：线程池适用于需要频繁创建和销毁线程的高并发场景，例如 Web 服务器、数据库连接池、并行计算任务等。

  1. 如何处理线程池中的异常？

​ ・ 答：可以使用 try-catch 块捕获线程中的异常，并记录日志或采取恢复措施，以确保线程池继续运行而不崩溃。

高级问题

  1. 如何调整线程池的参数（如核心线程数和最大线程数）以优化性能？

​ ・ 答：调整线程池参数需要根据具体应用的任务特性和系统资源进行调优。可以通过监控线程池的性能指标（如任务完成时间、线程利用率等）来确定合适的参数。

  1. 线程池中的线程如何保持存活？如何回收空闲线程？

​ ・ 答：线程池中的核心线程会一直存活，即使它们处于空闲状态。对于超过核心线程数的线程，可以设置存活时间，当这些线程在指定时间内空闲时会被回收。

  1. 如何实现任务优先级？

​ ・ 答：可以使用优先级队列（如优先级堆）来存储任务，确保高优先级任务优先执行。
