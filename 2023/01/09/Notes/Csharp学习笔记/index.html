# 4. 函数传参

1.ref 参数

正常传参的话，与 c 一致，不会改变调用的变量，c 中可以加上 & 解决这个问题，在 c# 中，则需要加上关键字，ref，对比如下：

c++

void Add(double a,double b,double &c)
{
  c = a + b;
}

c#

static void Add(double a,double b,ref double c)
{
  c = a + b;
}
//调用时也要加ref
Add(a,b,ref result);

加入关键字后变成引用传递

函数外必须赋值，函数内可以不赋值

2.out 参数

out 的功能是，可以让一个函数返回多个不同类型的值

在某一些情况下，很难去确定某个函数的返回值类型，可能是多个的，例如，int 类型的变量调用，但函数功能是求平均值，可能出现小数等情况，详细见下代码

public static void Main(string[] args)
{
  int a = 5;
  int b = 10;
  int m_max;
  int m_min;
  int m_sum;
  double m_avg;
  Number(a,b,out m_max,out m_min,out m_sum,out m_avg);
  Console.WriteLine("max:{0},min:{1},sum:{2},average:{3}",m_max,m_min,m_sum,m_avg);
  Console.ReadKey();
}
//功能，要求函数传入两个参数，返回最大、最小、和、平均值
static void Number(int a,int b,out int max,out int min,out int sum,out double average)
{
  max = a>b?a:b;
  min = a>b?b:a;
  sum = a + b;
  double sum2 = sum;
  average = sum2/2;
}
# 5. 函数递归

和 c 一样，简单提一下

从原理上来说，函数递归的实现是基于函数调用时的调用栈实现的，每次执行函数时，由于存在大量函数，因此需要通过栈来实现正确的执行顺序，通过这种特性，就可以实现一些功能，尤其是在动态规划的问题中。

例如下题 (JavaScript)：

var minPathSum = function(grid) {
    var width=grid[0].length;
    var height=grid.length;
    // const dp = Array.from(new Array(height), () => new Array(width).fill(1));
    var dp=[];
    for(let i=0;i<height;i++)
    dp[i]=[];
    dp[0][0]=grid[0][0];
     if(!width||!height) return 0;
     for(let i=1;i<height;i++)
     dp[i][0]=grid[i][0]+dp[i-1][0];
     for(let i=1;i<width;i++)
     dp[0][i]=grid[0][i]+dp[0][i-1];
     for(let i=1;i<height;i++)
     for(let j=1;j<width;j++)
     {
         dp[i][j]=Math.min(dp[i-1][j]+grid[i][j],dp[i][j-1]+grid[i][j]);
     }
     return dp[height-1][width-1];
};

# 9.StringBuilder

1. 功能说明

在使用字符串的过程中，当对一个字符串变量进行重复赋值时，会在内存中产生大量的垃圾数据，占据内存，当重复赋值的频率很高时，效率就会很低，原因是字符串重新赋值会产生新的内存空间并映射到变量，但原来的空间未被清理，这时使用 StringBuilder 就可以很有效地优化这一点，其使用的一直是同一片内存空间。

2. 使用方法

需要先引用

# 二。面向对象

# 1. 面向对象基础

面向对象指的是同样是解决一个问题或完成一个事项，只需要一个简单的操作（引用）就可以完成，例如开空调，只需要按下遥控器开启按键，空调就开始了运行，我们不需要知道空调的运行原理，只是触发了一个开关，就完成这个任务，依赖关系没有那么紧，这种就叫做面向对象。

不同于面向过程的高耦合，低内聚，面向对象是高内聚，低耦合的，复杂的内容和过程会封装在类中。

# 2. 继承

# 1. 基础知识

在类中，有父类和子类的概念，又称基类和派生类，基础使得类可以继承另一个类的特性，从而提高了代码的复用性。

# 2. 作用
  1. 使程序的创建和维护变得很容易；
  2. 有利于代码的重用和节省时间；
  3. 编写时候可以更快的理解和应用；
# 3. 特点
  1. 继承了基类 / 父类的属性和方法，子类可以继承父类的可继承属性、函数；继承时并非完全继承，只有可继承的类成员才可以被继承，主要使用访问修饰符进行设置；
  2. 继承实质上是对象与对象之间的关系；
  3. C# 继承只能实现单继承，每个类只能继承一个父类，一个父类可以拥有多个子类；
  4. 继承可以隔代继承，也就是子类下面还可以有子类；不允许相互继承 (循环继承);
  5. C# 不允许多重继承，但是可以实现多重接口继承；

# 4. 封装

封装: 即隐藏对象的属性和实现细节，仅对外公开接口，控制在程序中属性的读和修改的访问级别；将抽象得到的数据和行为（或功能）相结合，形成一个有机的整体，也就是将数据与操作数据的源代码进行有机的结合，形成 “类”，其中数据和函数都是类的成员。

封装的作用: 封装最主要的功能在于我们能修改自己的实现代码，而不用修改那些调用我们代码的程序片段。适当的封装可以让程式码更容易理解与维护，也加强了程式码的安全性，使不同类之间的相互影响减少到最低限度，进而增强数据的安全性和简化程序的编写工作。

封装的优点:

  1. 良好的封装能够减少耦合。
  2. 类内部的结构可以自由修改。
  3. 可以对成员变量进行更精确的控制。
  4. 隐藏信息，实现细节。

# 5. 虚方法

1. 什么是虚方法？

简单来说，就是在定义一个基类时，当某个类函数可能在其不同派生类中重写时，这个方法就应该定义为虚方法。

2. 作用

主要作用就是发挥了面向对象多态的特性，例如不同的派生类使用相同的方法时内部又存在部分差异，这时候可以重写该方法，使派生类有各种特性。

3. 使用方式

class Animal
{
    public string type;

    public virtual void Call()
    {
        Console.WriteLine("What am I?");
    }
}

class Frog : Animal
{
    public override void Call()
    {
        Console.WriteLine("呱呱呱呱");
    }
}

class Program
{
    static void Main(string[] args)
    {         
        Frog blue = new Frog();
        blue.type = "frog";
        blue.Call();
    }
}

# 6. 抽象类

1. 抽象类是什么？

在普通的类中，只要定义了方法就必须写出方法体，而在抽象类中不一定需要，因为本身就是一个抽象的概念，一些方法即使写了也无法定义具体内容，只能在派生类中定义，因此抽象类本身也无法实例化。

2. 抽象类的作用

在定义一个很抽象的大概念的类时，有些方法无法写出方法体，这时使用抽象类，即可不需要写出方法体，同时定义其他属性。

3. 如何使用

abstract class People
{
    public string name;
    public int age;
    public abstract void IntrouduceMyself();
}

absclass Student : People
{
    public override void IntrouduceMyself()
    {
        Console.WriteLine("Hello,everyone,I am a student from HFUT");
    }
}

class MathStu : Student
{
    public override void IntrouduceMyself()
    {
        Console.WriteLine("Hello,everyone,I am a student who learning Math " +
                          "from HFUT,my name is {0},my age is {1}",name,age);
    }
}

class Program
{
    static void Main(string[] args)
    {
        MathStu xyh = new MathStu();
        xyh.name = "Xu Yihang";
        xyh.age = 22;
        xyh.IntrouduceMyself();
    }
}

注意：

  1. 抽象类在定义时一定要加上修饰词 abstract。
  2. 抽象类的派生类一定要写出抽象类中方法的方法体。
  3. 派生类中写方法体时一定要加上修饰词 override。
  4. 抽象类无法直接实例化，使用时需要实例化其派生类，不过可以实例化派生类再向上转型达到间接实例化的效果。
  5. 从开发的角度来说，普通类应该尽可能的继承抽象类，并且抽象类也可以继承抽象类。

# 7. 接口

1. 什么是接口

接口，可以类比现实中的情况，例如，主板接口，电视机接口，显卡接口，从这些概念中可以理解，接口是某一种特定的类给出的一系列用于实现某些功能的渠道和方式。在 C# 中，接口默认都是抽象的，其中可以有属性，还有方法，但不能有方法体，只有方法的特征，没有方法的实现。

2. 接口的作用

简单来说就是，在保证封装性的同时更快更方便的提供一些方法。用来实现多态的特性。

3. 如何使用

interface ITest
{
    string name { get; set; }
    void Print();
}

interface ITest2
{
    void Dosomething();
}

class Animal : ITest
{
    //必须定义接口中的全部内容否则会报错
    public string name { get; set; }
    public void Print()
    {
        Console.WriteLine("嘿嘿，我是一只动物");
    }
}

class Test
{
    
}

//类只能继承一个父类，但可以有多个接口
class Frog : Animal , ITest2 
{
    //接口不需要修饰符就实现了函数的重写，体现了多态的特点
    public void Print()
    {
        Console.WriteLine("呱呱呱，我是一小青蛙");
    }

    public void Dosomething()
    {
        Console.WriteLine("我跳起来啦");
    }
}

class Program
{
    static void Main(string[] args)
    {
        Frog n1 = new Frog();
        n1.Print();
        n1.Dosomething();
    }
}

应用场景举例

例如银行账户

分为：普通会员，高级会员

普通会员：转账，取款，存款，查询余额，转账有手续费

高级会员：转账，取款，存款，查询余额，转账无手续费，免费的短信通知

这个时候，就可以把相同的部分做成一个接口，不同的部分单独做成另外的接口

# 8. 委托 (delegate)

1. 委托是什么？

委托，可以理解为函数指针 (c++ 中) 的升级版，调用函数时，可以通过函数名直接调用，也可以通过函数指针来间接调用，同样地，委托也可以实现间接调用。

2. 委托的作用

3. 如何使用

下面是关于委托的

class Calculator
{
    public void Report()
    {
        Console.WriteLine("I have 3 methods");
    }

    public int Add(int a,int b)
    {
        return a + b;
    }
    
    public int Sub(int a,int b)
    {
        return a - b;
    } 
}
class Program
{
    static void Main(string[] args)
    {
        int x = 200;
        int y = 300;
        int z = 0;
        Calculator Test = new Calculator();
        
        //定义委托
        Action action = new Action(Test.Report);
        action(); //输出 I have 3 methods

        Func<int, int, int> func1 = new Func<int, int, int>(Test.Add);
        Func<int, int, int> func2 = new Func<int, int, int>(Test.Sub);

        z = func1(x, y);
        Console.WriteLine(z);
        z = func2(x, y);
        Console.WriteLine(z);
    }
}
