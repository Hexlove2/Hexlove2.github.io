# 项目地址：

# https://github.com/Hexlove2/Linux_HttpServer

# 项目结构：

# 1. 互斥信号量类

使用 pthread 封装信号量，互斥锁，条件变量类，sem_wait

# 2. 线程池类

成员变量：

线程池数组，线程数，最大请求数，请求任务队列，停止标志，互斥锁，信号量

成员函数：

append 函数，添加任务，worker 函数，执行 run，run 函数，各个线程不断在任务队列里取任务并执行其处理函数，构造函数，创建线程等

# 3.http 请求类

需要完成的功能：
包括：接受请求数据，解析请求数据，根据不同请求完成需求，写到回复缓冲区并发生数据，处理连接状态

# 1. 处理请求

当 main 函数循环检测 IO 通知到的事件时，发现为 EPOLLIN 事件，这时会将这个 socket id 对应的请求类实例传给线程池队列，线程池取到该任务时就会调用该实例的 process 函数，这里就会分别执行以下步骤
1. 一次性读入数据到缓冲区，这是在 main 函数里就直接执行了
2. 解析请求头，请求行，请求体，使用状态机，此时 buffer 里已经读到了全部数据
3. 生成回应并传给客户端，这里用到了内存映射相关的系统调用，将 html 文件的内容写到请求体里，浏览器会解析到 200 ok 并发现为 html 类型

# 2. 初始化和关闭连接等

1. 初始化时需要设置好各个参数，设置端口复用，添加到 epoll 对象等

# 4.main 函数

1. 初始化，线程池相关，epoll 相关，socket 相关

2. 循环检测 epoll 已准备好的事件，并依次针对事件的种类进行判断处理，有时是连接请求，有时是 http 请求

3. 直接读写好数据并将其添加到任务队列，进行 http 请求的处理

# 5. 相关问题

1.epoll 相对于 poll 和 select 的优势？

epoll 是事件驱动的，而其他的是线性扫描，并且 epoll 可支持边缘触发搭配非阻塞 io，注意需要一次性读完数据，使用的是红黑树存储文件描述符，且支持的最大 fd 数很大

2. 还有相关的优化或扩展功能吗？

有在尝试手写一个定时器，通过定时器管理长期没有请求的客户端连接，将其断开，每一个定时器都有一个超时时间，每 5 秒会有一个 alarm 信号提醒检查定时器队列是否有超时，若出现超时，则调用注册好的回调函数
